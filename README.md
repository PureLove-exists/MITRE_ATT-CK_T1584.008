# Защита сетевого оборудования и внедрение концепции Zero Trust

Репозиторий содержит материалы научно-исследовательской статьи, посвященной защите плоскости управления (*Control Plane*) сетевых устройств от сетевых атак.

## 🎯 Цель проекта
Продемонстрировать уязвимость протокола Telnet на практике и настроить безопасный удаленный доступ по концепции Zero Trust.

## 🏗️ Стенд и параметры устройств
Эксперимент проводился в изолированной лабораторной среде:

| Устройство | Роль | Операционная система | IP-адрес |
| :--- | :--- | :--- | :--- |
| **Eltex ESR-15R** | Маршрутизатор | Eltex ESR-OS | `192.168.100.1/24` |
| **Cisco 1941** | Маршрутизатор | Cisco IOS | `19.103.27.1/24` |
| **MikroTik CRS326** | Коммутатор L3 | MikroTik RouterOS | `19.103.27.1/24` |
| **Kali Linux** | Хост атакующего | Linux (Debian) | `192.168.100.2; 10.19.103.60` |
| **Администратор** | Легитимный хост | Linux (Ubuntu) | `192.168.100.50; 10.19.103.50` |

---

## 🛡️ Реализованная защита
В ходе работы были настроены три разных подхода к безопасности:
1. **Eltex ESR:** Настройка зонального файрвола (ZBPFW) — доступ к роутеру разрешен только для IP администратора.
2. **Cisco 1941:** Защита виртуальных линий (VTY) с помощью списков доступа (ACL).
3. **MikroTik:** Ограничение доступа к службам через меню IP Services.
